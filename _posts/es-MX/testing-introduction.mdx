## Primero Que es whitebox testing?

Es una técnica en la que se comprueba cómo se prueba la estructura, el diseño y el código de nuestro software para verificar su entrada y salida. Esta técnica mejora el diseño, la usabilidad y la seguridad. 
Pero ¿por qué caja blanca? porque el código es visible para el probador, por lo que también se llama **Clear Box Testing**.

## Tecnicas White Box Testing 

Las siguientes son importantes técnicas de prueba de WhiteBox:

- Cobertura de estado de cuenta
- Cobertura de decisión
- Cobertura de Sucursales
- Cobertura de condiciones
- Cobertura de condiciones múltiples
- Cobertura de máquina de estado finito
- Cobertura de ruta
- Pruebas de flujo de control
- Pruebas de flujo de datos

Estos son algunos tipos de whitebox.

## Typos White Box Testing

Las pruebas White Box abarcan varios tipos de pruebas que se utilizan para evaluar la usabilidad de una aplicación, un bloque de código o un paquete de software específico.

- Pruebas Unitarias: A menudo es el primer tipo de prueba que se realiza en una aplicación. Las pruebas unitarias se realizan en cada unidad o bloque de código a medida que se desarrolla.
- Pruebas de fugas de memoria: las fugas de memoria son las principales causas de que las aplicaciones se ejecuten más lentamente.
